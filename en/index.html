new FileDragBox('#dragbox', {

        preview.innerHTML = '';

            img.title = image.name;

        });
    }
});

What is FileDragBox.js?

FileDragBox.js is a simple and minimalist library that allows you to turn an HTML element into a drag and drop container for receiving files, you can optionally attach the received files to an <input:file> to work with traditional forms.

How does it work?

Following the minimalist philosophy, FileDragBox.js does not interfere with the interface, you have all the freedom to give the design to your element where the files will be dragged and dropped, this library is only responsible for giving it the functionality, giving the element its virtues similar to an <input:file> element but simply by dragging the files into the box.

How to install it?

Shape #1. Downloading the 'FileDragBox.js' library file locally, or using the CDN and adding it to your html.

<script src="./lib/filedragbox.js"></script> <!-- IN LOCAL -->
<script src="//cdn.pkg/lib/filedragbox.js"></script> <!-- CDN -->

Shape #2. Using NPM.

> npm install filedragbox --save
import FileDragBox from './lib/filedragbox.js';

How to use it?

Step 1: Using it is very easy, if you download it locally or are using it with CDN, add the <script:src> with the library to your html.

<script src="//cdn.pkg/lib/filedragbox.js"></script> <!-- CDN -->

If you are using NPM, import the library in your JavaScript.

import FileDragBox from './lib/filedragbox.js';

Step 2: Add an element in your html (to which the file will be dragged and dropped), to this element add an 'id' or a 'class' to identify the element.

<div id="dragbox">
    Drag and Drop Files Here!
</div>

Step 3: Finally, create a new instance of the 'FileDragBox' class: new FileDragBox('selector', options). The constructor receives two arguments, the selector of the element to which the drag and drop is to be applied (in this example the selector is the id '#dragbox'), and as a second argument the options.

new FileDragBox('#dragbox', {
    accept: ['image/jpeg', 'image/png'],
    success: function (e, files) {
        console.log(files)
    }
});

Note: The 'success:' option receives an anonymous function which supplies two arguments to the function, the first is the 'event' and the second is the 'FileList' (file collection) of the files that have been dragged into the box, in the case of the example, those that have been dragged to the element '#dragbox' (in the case of the example, each time files are dragged to the element, the files "contained" in the box are displayed in the console, that is to say , the files that have been stored in the FileList).


The 'accept' option (optional), receives an array with the file types allowed in the drag and drop, if you drag a file of a type that is not in the array, it will not be added to the FileList.


Use without input:file

If you don't need to use a 'file' <input> associated with the drag and drop element, you can use an element to drag the files and via the success option you receive the dragged files and be done with them. what you need eg. send them to the backend or display them in the DOM, etc.

new FileDragBox('#dragbox', {
    success: function (e, files) {
        console.log(files)
    }
});

Important, success will be executed every time files are dragged and dropped onto the element (in this case the '#dragbox' element).


Use associated with an input:file

This option is perfect for working with traditional forms. To associate an <input:file> inside the drag and drop, you need to add the 'inputFile:' option, which receives a string with the selector (class or id) of the <input:file> you want to associate when dragging and dropping.

<div id="dragbox">
    Drag and Drop Images or <label for="file">Click Here!</label>
    <input type="file" name="files[]" id="file" multiple hidden>
</div>
new FileDragBox('#dragbox', {
    inputFile: '#file',
    success: function (e, files) {
        console.log(files);
    }
});

Now when you drag files into the element they will be added to the 'FileList' of the associated <input:file>.


If you submit the form in the traditional way, any files you drag into the drag and drop element will also be submitted with the associated <input:file>.


Every time files are dragged into the box, a 'change' event is also dispatched on the associated input, so in this case it's not strictly necessary to use 'success:', you could use myInput.addEventListener('change', (e) => {}) instead of 'success:'.

new FileDragBox('#dragbox', { inputFile: '#file' });
const myInput = document.querySelector('#dragbox');
myInput.addEventListener('change', (e) => {
    console.log('MyInput:', e.target.files);
})

Note: The 'success:' function will also fire when you add files via the associated <input:file>.

How to clear the file list?

To clear (reset) the list of files in the drag and drop element and associated input, you can use the clear() method.

const mydragbox = new FileDragBox('#dragbox', { inputFile: '#file' });
mydragbox.clear(); // Clear Files (items) in DataTransfer

Options

Option Description Default
inputFile Receives a string with the input selector (of type file) to associate with the drag and drop element. Null
(Without Associated Input)
accept Receive an array with the accepted file types ['type/format']:
  • ['image/jpeg', 'image/png'] (only accepts jpeg, jpg, png images)
  • ['image/*'] (only accepts images, in any format)
  • ['video/*'] (only accept videos)
  • etc.
 Null
(Any File)
success Receives a callback, which is supplied with two arguments, the event and the FileList (list with the added files).  Recibe un callback, al cual se le proporcionan dos argumentos, el event (evento) y el FileList (lista con los archivos agregados).

Example: success: (event, FileList) => console.log(FileList);

This is executed whenever files are dragged onto drag and drop items or items are added via the associated input.

Note: If I drag the files it will receive a 'drop' type event, if it is done through the associated input, the event will be of 'input' type.
 Null

Methods

Method Description
clear() Removes (resets) the list of files added to the FileList (removes the DataTrasfer and input.value elements).

You can use it when you need to remove/clean up the added files, after the form is submitted with JavaScript.

Note: If you submit the form the traditional way, this won't be necessary, as submitting it reloads the page and clears the form information.
